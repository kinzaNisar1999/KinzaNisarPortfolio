Bus Simulator Game

Role: Lead Programmer & Level Designer

Watch Gameplay Playlist

Gameplay Gallery

1. Project Overview

I developed the entire codebase and level design for this commercial simulation game. The game features 5 distinct modes (Career, Free Mode, Parking, Racing, and Challenge). I handled all programming, UI implementation, back-end integration, and optimization.

2. Key Technical Contributions

  • Core Gameplay: Implemented vehicle physics using RCC (Realistic Car Controller), fine-tuning suspension, torque, and handling for different vehicle types (Buses vs. Heavy Trucks).
  • AI System: Integrated RCC AI for the Racing Mode to create competitive enemy vehicles with overtaking and braking logic.
  • Level Design: Designed and built 20+ levels with varying environments, ensuring proper difficulty progression and terrain flow.
  • UI Architecture: Wired the entire UI flow (Main Menu, Garage, HUD, Pause, Game Over) and connected it to the game logic.
  • Backend & Monetization: Integrated Firebase Analytics to track user retention and AdMob Mediation for revenue.

3. Challenges & Solutions

  • Scalable Cutscenes: Creating unique passenger pick-up/drop-off animations for 20 levels was inefficient. I engineered a Generalized Cutscene Script that automatically handled character instantiation, movement, and boarding logic based on assigned waypoints.
  • Heavy Truck Physics: The new truck update with oversized tires caused flipping. I wrote a custom controller extension to override the center of mass and adjust suspension spring/damper rates dynamically based on speed.
  • Mobile Optimization: Addressed low FPS on older devices by implementing Occlusion Culling, baking lighting maps, and using texture compression to reduce memory footprint.

3D Animated Movie (FYP)

Role: Technical Artist

Watch Full Movie

Visuals

1. Technical Pipeline Overview

For my Final Year Project (Grade A), I co-produced a complete story-driven short film. Instead of traditional pre-rendering, we utilized Unity's Universal Render Pipeline (URP) to achieve near-real-time cinematic fidelity. My primary focus was establishing a robust character and asset pipeline between Daz 3D, Blender, and Unity.

2. Key Technical Contributions

  • Character Pipeline: Engineered a workflow using Daz 3D to generate age-variant morphs (Child vs. Adult) while maintaining consistent facial identity.
  • Cloth Simulation: Designed custom outfits in Marvelous Designer, simulating realistic fabric physics, and optimized mesh weights in Blender before Unity import.
  • Environment & Lighting: Constructed immersive scenes in Unity URP, utilizing a mix of baked lightmaps for static geometry and real-time lighting for characters.
  • Audio-Visual Sync: Manually synchronized voice-overs with facial animations directly within the Unity Timeline for precise lip-sync.

3. Challenges & Solutions

  • Facial Animation: Lacking paid motion capture tools, I developed a manual workflow using Unity Timeline to break down audio into phonemes and keyframe facial blendshapes.
  • Cloth Clipping: High-fidelity clothes clipped during fast movements. I created a cleanup workflow in Blender to bake physics simulations into animation frames before import.

Heavy Truck USA

Role: Unity Developer

Gameplay Gallery

1. Project Overview

I served as a core developer for this heavy cargo simulation game, designing specialized game modes that required unique handling logic. My primary focus was on vehicle physics stability for heavy loads (10+ tons) and dynamic terrain traversal.

2. Key Mechanic: Trailer Hitching System

I engineered a physics-based coupling system where the truck must manually align with the trailer's kingpin. Upon connection, a ConfigurableJoint is instantiated to link the rigidbodies, and the truck's mass/drag values are dynamically updated to simulate the massive added load.

3. Key Feature: Animal Transportation Mode

  • Live Cargo Logic: Designed a unique mode where players transport livestock. I implemented logic where rough driving (sudden braking/turning) agitates the animals, causing a score penalty or fail state.
  • Vehicle Physics: Adjusted suspension stiffness specifically for this mode to handle the shifting center of mass from moving live cargo.

4. Optimization

  • Terrain: Optimized large-scale terrain rendering using Terrain Chunking and LOD systems.
  • Scene Streaming: Implemented Additive Loading to load map sectors asynchronously, reducing memory usage on low-end devices.
